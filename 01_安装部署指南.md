## 1. 前置环境要求

| 工具 | 最低版本 | 推荐版本 | 验证命令 |

## 2. 迁移项目到新电脑

### 2.1 需要复制的文件

将 `smart-store/` 文件夹复制到新电脑，**但排除以下自动生成的目录/文件**：

| 排除项 | 原因 | 大小 |
|--------|------|------|
| `frontend/node_modules/` | npm install 会重新生成 | ~200MB |
| `backend/__pycache__/` 及子目录中的 `__pycache__/` | Python 运行时自动生成 | 几MB |
| `backend/smart_store.db` | seed_data.py 会重新生成 | ~1MB |

> **快速复制方法（Windows）**：
> 1. 复制整个 `smart-store/` 文件夹
> 2. 在新电脑上删除 `frontend/node_modules/` 文件夹（如果复制了的话）
> 3. 删除 `backend/smart_store.db`（如果复制了的话）
>
> 或者用压缩软件打包时排除 `node_modules` 即可，其他文件总共只有几MB。

### 2.2 项目目录结构

```
smart-store/
├── backend/                ← Python 后端
│   ├── app/
│   │   ├── main.py         ← 应用入口
│   │   ├── config.py       ← 配置（数据库、AI Key）
│   │   ├── database.py     ← 数据库连接
│   │   ├── models/         ← 数据模型（7个表）
│   │   ├── routers/        ← API路由（7个模块）
│   │   ├── schemas/        ← 请求/响应模型
│   │   └── services/       ← 业务逻辑层
│   ├── seed_data.py        ← 数据初始化脚本
│   └── requirements.txt    ← Python依赖清单
└── frontend/               ← Vue 3 前端
    ├── src/
    │   ├── api/            ← 后端接口封装
    │   ├── components/     ← 公共组件（Layout）
    │   ├── router/         ← 路由+权限守卫
    │   ├── stores/         ← Pinia状态管理
    │   └── views/          ← 页面组件（8个）

    └── package.json        ← Node依赖清单
```

### 2.3 在新电脑上的安装步骤

确保新电脑已安装前置环境（见第1节），然后依次执行第3节和第4节的安装步骤即可。

完整流程总结：
```bash
# 1. 安装后端依赖
cd smart-store/backend
pip install -r requirements.txt

# 2. 初始化数据库
python seed_data.py

# 3. 安装前端依赖（首次需要几分钟）
cd ../frontend
npm install

# 4. 启动后端（新开一个终端）
cd smart-store/backend
uvicorn app.main:app --reload --port 8000

# 5. 启动前端（新开一个终端）
cd smart-store/frontend
npm run dev
```

## 3. 后端安装步骤

```bash
cd smart-store/backend

```

- `fastapi` - Web 框架
- `uvicorn` - ASGI 服务器
- `sqlalchemy` - ORM

## 4. 前端安装步骤

- `vue-router` - 路由

### 4.2 启动前端开发服务器

```bash
cd smart-store/frontend
npm run dev
```

启动成功标志：
```
  VITE v5.x.x  ready in xxx ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
```

### 4.3 API 代理说明

前端 `vite.config.js` 已配置代理，将 `/api` 前缀的请求转发到后端：

```javascript
proxy: {
  '/api': {
    target: 'http://localhost:8000',
    changeOrigin: true,
    rewrite: (path) => path.replace(/^\/api/, '')
  }
}
```

即前端调用 `/api/auth/login` 会自动转发到 `http://localhost:8000/auth/login`。

## 6. 一键启动脚本

将以下内容保存为 `start.bat`，放在 `smart-store/` 根目录下：

```bat
@echo off
echo ============================================
echo   优尚服饰门店运营管理系统 - 一键启动
echo ============================================
echo.

echo [1/3] 初始化数据库...
cd /d %~dp0backend
python seed_data.py
echo.

echo [2/3] 启动后端服务（端口 8000）...
start "Smart-Store Backend" cmd /k "cd /d %~dp0backend && uvicorn app.main:app --reload --port 8000"
timeout /t 3 /nobreak >nul
echo   后端已启动
echo.

echo [3/3] 启动前端服务（端口 5173）...
start "Smart-Store Frontend" cmd /k "cd /d %~dp0frontend && npm run dev"
timeout /t 5 /nobreak >nul
echo   前端已启动
echo.

echo ============================================
echo   系统启动完成！
echo   前端地址: http://localhost:5173
echo   后端地址: http://localhost:8000
echo ============================================
pause
```

使用方法：双击 `start.bat` 即可，会自动打开两个命令行窗口分别运行前后端。

## 7. 常见问题排查

### Q1：端口被占用

**现象**：启动时报 `Address already in use`

**解决**：
```bash
# 查看占用 8000 端口的进程
netstat -ano | findstr :8000

# 杀掉对应进程（替换 PID）
taskkill /F /PID <PID>
```

或者换端口启动：
```bash
uvicorn app.main:app --reload --port 8001
```

> 注意：如果后端换了端口，前端 `vite.config.js` 中的 proxy target 也需要同步修改。

### Q2：pip install 报错

**现象**：安装 `cryptography` 或 `bcrypt` 失败

**解决**：
```bash
# 升级 pip
python -m pip install --upgrade pip

# 如果仍然失败，尝试安装预编译包
pip install --only-binary :all: cryptography bcrypt
```

### Q3：npm install 很慢或失败

**解决**：切换 npm 镜像源：
```bash
npm config set registry https://registry.npmmirror.com
npm install
```

### Q4：数据库重置

如果测试数据被弄乱，需要重置：
```bash
cd smart-store/backend
# 删除数据库文件
del smart_store.db
# 重新生成
python seed_data.py
```

### Q5：前端页面空白或报跨域错误

**检查**：
1. 确认后端正在运行（访问 http://localhost:8000 有响应）
2. 确认前端的 vite 代理配置正确
3. 确认前端 API 请求路径以 `/api` 开头

### Q6：登录提示"用户名或密码错误"

**可能原因**：数据库未初始化或数据丢失

**解决**：重新运行 `python seed_data.py`

## 8. 如何切换到 MySQL（如需要）

当前项目使用 SQLite 便于开发测试，如需切换到 MySQL：

### 步骤 1：安装 MySQL 驱动

```bash
pip install pymysql
```

### 步骤 2：修改数据库配置

编辑 `backend/app/config.py`：

```python
# 原配置（SQLite）
# DB_URL = "sqlite:///./smart_store.db"

# MySQL 配置
DB_URL = "mysql+pymysql://用户名:密码@localhost:3306/smart_store?charset=utf8mb4"
```

### 步骤 3：创建 MySQL 数据库

```sql
CREATE DATABASE smart_store CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 步骤 4：重新初始化数据

```bash
python seed_data.py
```

> 由于使用 SQLAlchemy ORM，代码层无需任何修改，只需更换连接字符串即可。
